# BOJ 1966 - 프린터 큐

## 문제 설명
- [문제 링크](https://www.acmicpc.net/problem/1966)
- 프린터기에서 문서의 중요도에 따라 인쇄 순서를 결정하는 문제이다.
- 현재 문서보다 중요도가 높은 문서가 하나라도 뒤에 있다면, 현재 문서를 가장 뒤로 보낸다.
- 특정 위치 `M`에 있는 문서가 몇 번째로 인쇄되는지 구해야 한다.

## 아이디어
- (중요도, 초기 인덱스)를 튜플로 묶어 `deque`에 담는다.
- 큐에서 하나를 꺼냈을 때, 남은 큐 안에 현재보다 높은 중요도가 있는지 확인한다.
- 더 높은 중요도가 있다면 다시 큐의 뒤에 넣고, 없다면 인쇄 횟수를 카운트한다.
- 인쇄된 문서의 원래 인덱스가 찾고자 하는 `M`이라면 반복을 종료한다.

## 시간복잡도
- 테스트 케이스 `T`, 문서 개수 `N` (최대 100).
- 각 문서를 처리할 때마다 나머지 문서의 중요도를 검사하므로 한 케이스당 `O(N^2)`이다.
- 전체 시간복잡도는 `O(T * N^2)`로, 주어진 제한 내에서 충분히 해결 가능하다.

## 풀이 과정
1. 테스트 케이스 개수를 입력받는다.
2. 각 케이스마다 `N`(문서 개수)과 `M`(타겟 인덱스)을 입력받는다.
3. 문서들의 중요도를 입력받아 `(중요도, 인덱스)` 형태의 튜플로 큐를 구성한다.
4. 큐가 빌 때까지 다음 과정을 반복한다:
   - 큐의 앞부분에서 문서를 하나 꺼낸다.
   - 큐에 남은 문서들 중 더 큰 중요도를 가진 문서가 있는지 확인한다.
   - 있다면 현재 문서를 큐의 맨 뒤로 다시 보낸다.
   - 없다면 해당 문서를 출력(카운트 증가)하고, 해당 문서의 인덱스가 `M`인지 확인한다.
   - 인덱스가 `M`이라면 카운트된 값을 출력하고 해당 케이스를 종료한다.

## 코드 설명
```python
import sys
from collections import deque
input = sys.stdin.readline

T = int(input())
for _ in range(T) :
    N, M = map(int, input().split())
    prior = list(map(int,input().split()))
    # (중요도, 원래 인덱스) 형태로 큐 초기화
    q = deque([(p,i) for i, p in enumerate(prior)])
    num = 0
    while q :
        now, idx = q.popleft()
        # 현재 문서보다 중요도가 높은 문서가 뒤에 있는지 확인
        if q and now < max(p for p,_ in q) :
            q.append((now,idx))
        else :
            num += 1 # 인쇄
            if idx == M :
                break
    print(num)
```
- `collections.deque`를 사용하여 팝과 푸시 연산을 효율적으로 처리했다.
- `max()` 함수를 이용해 큐 내의 최대 중요도를 확인했다.
