# JUNGOL 1111

## 문제 설명
- [문제 링크](https://jungol.co.kr/problem/1111)
- `N x N` 크기의 산 높이 정보와 목표 지점의 좌표가 주어진다.
- 산 바깥에서 출발해 목표 지점까지 이동할 때 필요한 최소 비용을 구하는 문제이다.
- 상하좌우로 이동할 수 있으며, 이동 전 높이를 `a`, 이동 후 높이를 `b`라고 하면 비용은 다음과 같다.
    - `a < b`: `(b - a)^2`
    - `a > b`: `a - b`
    - `a == b`: `0`

## 아이디어
- 각 칸을 정점, 상하좌우 이동을 간선으로 보면 가중치가 있는 그래프가 된다.
- 이동 비용은 항상 0 이상이므로 다익스트라 알고리즘을 사용할 수 있다.
- 산 바깥을 높이 0인 테두리로 확장해 `(0, 0)`에서 출발한다.
- 우선순위 큐에는 `{현재까지의 비용, 행, 열}`을 저장하고, 가장 비용이 작은 위치부터 탐색한다.

## 시간복잡도
- 정점 수는 확장된 격자 기준으로 `(N + 2) x (N + 2)`개이다.
- 각 정점에서 최대 4개의 방향을 확인한다.
- 우선순위 큐를 사용하는 다익스트라 알고리즘이므로 전체 시간복잡도는 `O(N^2 log N)`이다.

## 풀이 과정
1. 산의 크기 `n`과 목표 지점 `(top_i, top_j)`를 입력받는다.
2. 산 바깥 테두리를 포함하기 위해 `n + 2` 크기의 2차원 배열을 만든다.
3. 실제 산 높이는 `1`부터 `n`까지의 인덱스에 저장하고, 바깥 테두리는 높이 `0`으로 둔다.
4. 최단 비용 배열 `dist`를 무한대로 초기화한다.
5. 시작점 `(0, 0)`의 비용을 `0`으로 두고 우선순위 큐에 넣는다.
6. 큐에서 현재 비용이 가장 작은 칸을 꺼낸다.
7. 이미 더 작은 비용으로 방문한 칸이면 넘어간다.
8. 상하좌우 다음 칸이 확장된 배열 범위 안에 있으면 이동 비용을 계산한다.
9. 더 작은 비용으로 다음 칸에 도착할 수 있으면 `dist`를 갱신하고 큐에 넣는다.
10. 탐색이 끝나면 `dist[top_i][top_j]`를 출력한다.

## 코드 설명
```cpp
int getCost(int a, int b) {
    if (a < b) return (b-a)*(b-a);
    else if (a > b) return a-b;
    else return 0;
}
```
- 현재 높이 `a`에서 다음 높이 `b`로 이동할 때 필요한 비용을 계산한다.
- 올라갈 때는 높이 차이의 제곱, 내려갈 때는 높이 차이, 같은 높이일 때는 0을 반환한다.

```cpp
dist[0][0] = 0;
pq.push({0,0,0});

while(!pq.empty()) {
    auto [cost, r, c] = pq.top();
    pq.pop();
    if(cost > dist[r][c]) continue;

    for(int i = 0 ; i < 4 ; ++i) {
        int nr = r + dr[i];
        int nc = c + dc[i];
        if (0 <= nr && nr <= n+1 && 0 <= nc && nc <= n+1) {
            int nextCost = cost + getCost(mount[r][c], mount[nr][nc]);
            if (nextCost < dist[nr][nc]) {
                dist[nr][nc] = nextCost;
                pq.push({nextCost,nr,nc});
            }
        }
    }
}
```
- `dist[r][c]`는 시작점에서 `(r, c)`까지 이동하는 최소 비용이다.
- `priority_queue`를 최소 힙처럼 사용하기 위해 `greater<tuple<int,int,int>>`를 사용한다.
- 현재 저장된 최단 비용보다 큰 값이 큐에서 나오면 이미 처리할 필요가 없는 경로이므로 건너뛴다.
- 상하좌우로 이동하면서 더 작은 비용이 발견되면 최단 비용을 갱신한다.
