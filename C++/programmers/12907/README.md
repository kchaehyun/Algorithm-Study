# Programmers 12907 - 거스름돈

## 문제 설명
- [문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/12907)
- 거슬러 주어야 하는 금액 `n`과 사용할 수 있는 화폐 단위가 담긴 배열 `money`가 주어진다.
- 주어진 화폐를 사용하여 금액 `n`을 만드는 경우의 수를 구한다.
- 화폐를 사용하는 순서만 다른 경우는 같은 경우로 취급한다.
- 결과는 `1,000,000,007`로 나눈 나머지를 반환해야 한다.

## 아이디어
- `dp[i]`를 현재까지 확인한 화폐들로 금액 `i`를 만드는 경우의 수라고 정의한다.
- 아무 화폐도 사용하지 않고 금액 `0`을 만드는 경우가 하나 있으므로 `dp[0] = 1`로 시작한다.
- 각 화폐 단위 `m`에 대해 `m`부터 `n`까지 금액을 오름차순으로 순회한다.
- `dp[i - m]`의 각 경우에 화폐 `m`을 하나 추가하면 금액 `i`를 만들 수 있으므로 `dp[i] += dp[i - m]`으로 갱신한다.
- 화폐를 바깥 반복문에서 하나씩 처리하므로 같은 화폐 조합을 순서만 바꾸어 중복 계산하지 않는다.

## 시간복잡도
- 화폐 종류의 수를 `M`이라고 할 때 각 화폐마다 최대 `n`개의 금액을 확인하므로 O(M * N)이다.
- 크기가 `n + 1`인 DP 배열을 사용하므로 공간복잡도는 O(N)이다.

## 풀이 과정
1. 금액별 경우의 수를 저장할 `dp` 배열을 `n + 1` 크기로 만든다.
2. 금액 `0`을 만드는 경우의 수인 `dp[0]`을 `1`로 초기화한다.
3. `money`에서 화폐 단위 `m`을 하나씩 가져온다.
4. 금액 `m`부터 `n`까지 오름차순으로 순회하며 `dp[i - m]`을 `dp[i]`에 더한다.
5. 모든 화폐를 처리한 뒤 `dp[n]`을 반환한다.

## 코드 설명
```cpp
int solution(int n, vector<int> money) {
    vector<int> dp(n+1, 0);
    dp[0] = 1;
    for(int m : money) {
        for(int i = m ; i < n+1 ; ++i) {
            dp[i] += dp[i-m];
        }
    }
    return dp[n];
}
```
- `dp[0] = 1`은 현재 화폐를 사용해 그 화폐의 금액을 처음 만드는 경우를 계산하기 위한 기준값이다.
- 금액을 오름차순으로 순회하므로 같은 화폐를 여러 번 사용하는 경우도 현재 반복에서 누적된다.
- 화폐 단위별로 경우의 수를 누적하기 때문에 화폐를 고르는 순서가 다른 같은 조합은 중복 계산되지 않는다.
- 문제 조건에 맞게 안전하게 구현하려면 덧셈할 때마다 `1,000,000,007`로 나머지 연산을 해야 한다. 현재 코드는 이 연산이 없어 경우의 수가 커지면 `int` 오버플로가 발생할 수 있다.
