# Programmers 12987 - 숫자 게임

## 문제 설명
- [문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/12987)
- A팀과 B팀이 각각 자연수가 적힌 카드를 한 장씩 낸다.
- 두 카드의 숫자를 비교해 B팀의 숫자가 A팀의 숫자보다 크면 B팀이 승점 1점을 얻는다.
- A팀이 낼 카드의 순서 `A`와 B팀이 가진 카드 `B`가 주어진다.
- B팀이 얻을 수 있는 최대 승점을 반환한다.

## 아이디어
- A팀과 B팀의 카드를 모두 오름차순으로 정렬한다.
- B팀은 가능한 한 작은 카드로 A팀의 현재 가장 작은 카드를 이기는 것이 유리하다.
- `B[j]`가 `A[i]`보다 크다면, `B[j]`는 `A[i]`를 이기는 데 사용할 수 있는 가장 작은 후보이므로 둘을 매칭하고 승점을 1점 얻는다.
- `B[j]`가 `A[i]`보다 작거나 같다면, 현재 B 카드는 `A[i]`뿐 아니라 이후의 더 큰 A 카드도 이길 수 없으므로 버린다.
- 이렇게 두 배열을 앞에서부터 훑으면 B팀이 이길 수 있는 매칭 수를 최대로 만들 수 있다.

## 시간복잡도
- 두 배열을 정렬하는 데 O(N log N)이 걸린다.
- 투 포인터로 한 번 순회하는 데 O(N)이 걸린다.
- 전체 시간복잡도는 O(N log N)이다.
- 정렬을 제외하고 추가로 사용하는 공간은 O(1)이다.

## 풀이 과정
1. `A`와 `B`를 오름차순으로 정렬한다.
2. `A`의 현재 비교 위치를 `i`, `B`의 현재 비교 위치를 `j`로 둔다.
3. `A[i] < B[j]`이면 B팀이 이길 수 있으므로 `answer`를 1 증가시킨다.
4. 이긴 경우 두 카드를 모두 사용했으므로 `i`와 `j`를 모두 1 증가시킨다.
5. 이기지 못한 경우 B팀의 현재 카드는 사용할 수 없으므로 `j`만 1 증가시킨다.
6. 둘 중 한 배열을 모두 확인할 때까지 반복한 뒤 `answer`를 반환한다.

## 코드 설명
```cpp
int solution(vector<int> A, vector<int> B) {
    sort(A.begin(), A.end());
    sort(B.begin(), B.end());
    if(A.front() > B.back()) return 0;
    
    int answer = 0;
    int i = 0, j = 0;
    
    while(i < A.size() && j < B.size()) {
        if(A[i] < B[j]) {
            ++i;
            ++j;
            ++answer;
        }
        else ++j;
    }
    
    return answer;
}
```
- `sort`로 두 배열을 정렬해 작은 카드부터 비교한다.
- `i`는 아직 이기지 못한 A팀 카드 중 가장 작은 카드를 가리킨다.
- `j`는 아직 사용하지 않은 B팀 카드 중 가장 작은 카드를 가리킨다.
- `A[i] < B[j]`이면 현재 B 카드로 현재 A 카드를 이길 수 있으므로 승점을 얻고 두 포인터를 모두 이동한다.
- 그렇지 않으면 현재 B 카드는 어떤 방식으로도 현재 A 카드를 이길 수 없으므로 `j`만 이동한다.
- `A.front() > B.back()`인 경우 B팀의 가장 큰 카드도 A팀의 가장 작은 카드보다 작으므로 즉시 `0`을 반환한다.
