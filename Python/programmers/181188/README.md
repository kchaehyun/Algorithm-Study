# Programmers 181188 - 요격 시스템

## 문제 설명
[문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/181188)

각 폭격 미사일의 시작 지점(s)과 끝 지점(e)이 주어질 때, 모든 미사일을 요격하기 위해 필요한 최소한의 요격 미사일 수를 구하는 문제입니다. 요격 미사일은 실수 지점 $x$에서 발사하면 $s < x < e$인 모든 폭격 미사일을 요격할 수 있습니다.

## 아이디어
- 전형적인 **그리디(Greedy)** 문제로, 활동 선택 문제(Activity Selection Problem)와 유사하게 접근할 수 있습니다.
- 폭격 미사일의 **종료 지점(e)**을 기준으로 오름차순 정렬합니다.
- 현재 발사한 요격 미사일의 위치보다 늦게 시작하는 미사일이 나타나면, 새로운 요격 미사일을 해당 미사일의 종료 지점 직전에 발사합니다.

## 시간복잡도
- 미사일의 개수를 $N$이라 할 때:
- 정렬: $O(N \log N)$
- 리스트 순회: $O(N)$
- 전체 시간복잡도: $O(N \log N)$
- $N \le 500,000$이므로 제한 시간 내에 충분히 해결 가능합니다.

## 풀이 과정
1. 입력받은 `targets`를 종료 지점(`x[1]`)을 기준으로 오름차순 정렬합니다.
2. 마지막으로 발사한 요격 미사일의 위치를 나타내는 `end` 변수를 0으로 초기화합니다.
3. 정렬된 미사일들을 하나씩 확인합니다:
    - 현재 미사일의 시작 지점(`s`)이 `end`보다 크거나 같으면, 현재의 요격 미사일로는 이 미사일을 요격할 수 없습니다.
    - 따라서 새로운 요격 미사일을 발사하고(`answer += 1`), `end`를 현재 미사일의 종료 지점(`e`)으로 갱신합니다.
4. 모든 미사일을 확인한 후 `answer`를 반환합니다.

## 코드 설명
```python
def solution(targets):
    answer = 0
    end = 0
    # 종료 지점 기준으로 정렬
    targets.sort(key = lambda x: x[1])
    
    for s, e in targets :
        # 현재 요격 미사일의 위치가 새로운 미사일의 시작점보다 작거나 같으면
        # (개구간이므로 s와 같은 경우에도 새로운 미사일이 필요함)
        if s >= end:
            answer += 1
            end = e # 새로운 요격 미사일의 위치(종료 지점) 갱신
    return answer
```
