# BOJ 3986 - 좋은 단어

## 문제 설명
- [문제 링크](https://www.acmicpc.net/problem/3986)
- 단어 위로 같은 글자끼리 선을 그었을 때, 선이 교차하지 않도록 모든 글자를 짝지을 수 있으면 좋은 단어이다.
- 주어진 단어들 중 좋은 단어의 개수를 구하는 문제이다.

## 아이디어
- 인접한 같은 문자는 서로 짝이 되어 사라질 수 있다.
- 스택의 마지막 문자와 현재 문자가 같으면 짝을 이룬 것이므로 스택에서 제거한다.
- 다르면 현재 문자를 스택에 넣는다.
- 단어를 모두 확인한 뒤 스택이 비어 있다면 모든 문자가 올바르게 짝지어진 것이므로 좋은 단어이다.

## 풀이 과정
1. 단어의 개수 `N`을 입력받는다.
2. 각 단어마다 빈 스택을 만든다.
3. 단어의 문자를 왼쪽부터 하나씩 확인한다.
4. 스택이 비어 있지 않고 현재 문자가 스택의 마지막 문자와 같으면 `pop`한다.
5. 그렇지 않으면 현재 문자를 스택에 넣는다.
6. 단어를 모두 처리한 뒤 스택이 비어 있으면 좋은 단어 개수를 1 증가시킨다.
7. 최종 개수를 출력한다.

## 코드 설명
```python
import sys
input = sys.stdin.readline

N = int(input())
answer = 0
for _ in range(N) : 
    arr = input().strip()
    stack = []
    for a in arr :
        if stack and a == stack[-1] :
            stack.pop()
        else:
            stack.append(a)
    if not stack:
        answer += 1

print(answer)
```
- `stack[-1]`은 아직 짝을 찾지 못한 가장 최근 문자를 의미한다.
- 현재 문자와 `stack[-1]`이 같으면 두 문자는 짝이 되므로 제거한다.
- 문자를 모두 처리한 뒤 스택이 비어 있으면 모든 문자가 짝을 이룬 상태이다.
- 따라서 `not stack`인 경우에만 `answer`를 증가시킨다.
