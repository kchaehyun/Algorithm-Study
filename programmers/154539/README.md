# Programmers 154539 - 뒤에 있는 큰 수 찾기

## 문제 설명
- [문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/154539)
- 정수로 이루어진 배열 `numbers`가 주어질 때, 각 원소에 대해 자신보다 뒤에 있는 숫자들 중 자신보다 크면서 가장 가까이 있는 숫자인 '뒷 큰수'를 찾는 문제이다.
- 뒷 큰수가 존재하지 않는 원소는 -1을 담는다.

## 아이디어
- 배열의 길이가 최대 1,000,000이므로 이중 반복문을 사용한 `O(N^2)` 풀이는 시간 초과가 발생한다.
- **스택(Stack)**을 활용하여 아직 뒷 큰수를 찾지 못한 인덱스들을 관리함으로써 `O(N)`의 시간 복잡도로 해결할 수 있다.
- 현재 숫자가 스택의 TOP에 해당하는 인덱스의 숫자보다 크다면, 현재 숫자가 해당 인덱스의 뒷 큰수가 된다.

## 시간복잡도
- 배열의 모든 원소를 한 번씩 순회하며, 각 인덱스는 스택에 한 번 들어갔다가 한 번 나오게 된다.
- 따라서 전체 시간 복잡도는 `O(N)`이다.

## 풀이 과정
1. `numbers`와 같은 길이의 `answer` 배열을 -1로 초기화한다.
2. 아직 뒷 큰수를 찾지 못한 인덱스를 담을 빈 스택 `s`를 생성한다.
3. `numbers` 배열을 순회하며 다음 과정을 반복한다:
   - 스택이 비어있지 않고, 현재 숫자 `n`이 `numbers[스택의 TOP 인덱스]`보다 크면:
     - 스택에서 인덱스를 꺼내(`pop`), 해당 인덱스의 `answer` 값을 `n`으로 업데이트한다.
     - 이 과정을 조건이 만족하는 동안 반복한다 (현재 숫자가 여러 인덱스의 뒷 큰수가 될 수 있음).
   - 현재 인덱스 `i`를 스택에 추가한다.
4. 모든 순회가 끝나면 `answer` 배열을 반환한다.

## 코드 설명
```python
def solution(numbers):
    # 모든 원소에 대해 뒷 큰수가 없는 경우인 -1로 초기화
    answer = [-1] * len(numbers)
    # 뒷 큰수를 아직 찾지 못한 인덱스를 저장할 스택
    s = []
    
    for i, n in enumerate(numbers):
        # 스택에 인덱스가 있고, 현재 숫자가 스택 TOP 인덱스의 숫자보다 큰 경우
        # 즉, 현재 숫자가 스택에 저장된 인덱스의 '뒷 큰수'인 경우
        while s and numbers[s[-1]] < n:
            idx = s.pop()
            answer[idx] = n
        # 현재 인덱스를 스택에 추가 (자신의 뒷 큰수를 찾기 위해 대기)
        s.append(i)            
            
    return answer
```
- 스택에는 숫자가 아닌 **인덱스**를 저장하여 `answer` 배열의 해당 위치를 바로 업데이트할 수 있도록 한다.
- `while` 루프를 통해 현재 숫자가 이전의 여러 숫자들에 대한 뒷 큰수가 되는 경우를 모두 처리한다.
- 뒷 큰수를 찾지 못한 인덱스들은 스택에 남아있게 되며, `answer`의 초기값인 -1을 유지하게 된다.
